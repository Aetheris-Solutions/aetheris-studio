Overview

Cookies and browser storage are not the same thing

Cookies are small values a website or service can ask a browser to send with later requests. Local storage can retain information for a site across browser sessions, while session storage normally lasts for the current tab or browsing session. Similar technologies can also use scripts, pixels, device signals or in-memory values.

This notice covers those technologies together so that the actual behaviour of the site is easier to understand. The related handling of personal data is described in the privacy notice.

Security

Cloudflare Turnstile

The final step of the qualification form loads Cloudflare Turnstile from challenges.cloudflare.com. Turnstile uses browser, device, network and interaction signals to distinguish legitimate requests from automated abuse. It may use cookies or similar storage needed to provide that security function.

The browser receives a short-lived token. Aetheris's server sends that token and, where available, the connecting IP address to Cloudflare for verification. If verification fails, the server does not write the request to Attio. Turnstile is configured as a security control, not as an advertising tool.

Cloudflare says Turnstile processes signals including IP address, TLS fingerprint, user-agent header, site key and origin solely for bot detection and improvement. See Cloudflare's Turnstile Privacy Addendum. Blocking this necessary control can prevent the protected form from being sent; email remains available as an alternative.

Measurement

Analytics that runs only after opt-in

The qualification interface creates a small allowlisted set of lifecycle events in the page's in-memory dataLayer, such as form view, step completion, submission and result. The event payload deliberately excludes names, email addresses, companies, store URLs, free text, campaign values and CRM identifiers.

The site uses Google Consent Mode v2 in its basic form. Google Tag Manager, Google Analytics 4 and Microsoft Clarity are not downloaded and receive no optional measurement request until you actively accept analytics. A rejection or the close control leaves analytics_storage, ad_storage, ad_user_data and ad_personalization denied. Advertising and personalisation technologies are not active.

On the canonical Production host, the current GTM container can load GA4 and Clarity after opt-in. Public Cloudflare Pages preview hostnames keep all three services offline even after an analytics choice. GA4 measures page use; Clarity analyses consented rendered-page interactions, clicks and scrolls through session reconstruction and heatmaps. The site prepares a non-PII set of qualification lifecycle events in the data layer, but the current published container does not yet forward them as separate GA4 events. Aetheris must not pass contact details, free-text form content or CRM identifiers into either analytics service.

Consent is the legal basis under Article 6(1)(a) GDPR and Article 122 of the Italian Privacy Code. The analytics container must be re-inventoried whenever its tags or destinations change.

Controls

How to manage browser data

You can use your browser to inspect, block or remove cookies and site storage. The exact controls depend on the browser and device. Common options include:

  • clearing cookies and site data for aetherisstudio.com;
  • blocking third-party cookies or limiting cross-site tracking;
  • using private browsing for a session that should not retain local storage after it closes; and
  • using browser developer or privacy tools to inspect values stored for the site.

The banner gives equally prominent choices to reject non-essential technologies or accept analytics; “Preferences” opens the category control and the close button persists the denied default. No box is preselected and continuing to scroll is not treated as consent.

Use Cookie choices in the footer at any time to grant or withdraw analytics. Withdrawal updates consent to denied, removes first-party GA/Clarity cookies that the Aetheris origin can access, clears first-touch attribution and reloads the page so the tag runtime is no longer present. Microsoft-domain third-party cookies cannot be deleted by the Aetheris domain; browser controls can remove them.

Removing the hero session value may cause the opening sequence to play again. Blocking Turnstile or its required scripts can prevent the protected qualification form from being sent; you can still contact Aetheris at info@aetherisstudio.com.
