Overview

Cookies and browser storage are not the same thing

Cookies are small values a website or service can ask a browser to send with later requests. Local storage can retain information for a site across browser sessions, while session storage normally lasts for the current tab or browsing session. Similar technologies can also use scripts, pixels, device signals or in-memory values.

This notice covers those technologies together so that the actual behaviour of the site is easier to understand. The related handling of personal data is described in the privacy notice.

Current inventory

What the preview build stores

Technology Purpose Duration or control
aetheris-production-hero-intro-v1
Session storage		 Remembers that the opening visual sequence has been seen, so it is not replayed unnecessarily in the same browsing session. Normally ends with the browser tab or session. It can be removed through browser site-data controls.
aetheris_studio_attribution_v1
Local storage		 Remembers the first campaign/source touch used to contextualise a qualification request. It can include UTM values, ad click IDs, a sanitised landing path, a sanitised referrer and a timestamp. The current code sets no automatic expiry. It remains until site data is cleared or the implementation changes. The form still works if storage is unavailable.
Turnstile widget
Cloudflare security technology		 Assesses whether a form interaction is abusive and produces the short-lived token the server must verify before writing to the CRM. Controlled by Cloudflare's security service and browser/site-data settings. See the Turnstile section below.

The first-touch attribution record is sent to Aetheris only when you submit the qualification form. Stored landing and referrer URLs omit query strings and fragments; campaign parameters and recognised click identifiers are captured as separate fields.

Security

Cloudflare Turnstile

The final step of the qualification form loads Cloudflare Turnstile from challenges.cloudflare.com. Turnstile uses browser, device, network and interaction signals to distinguish legitimate requests from automated abuse. It may use cookies or similar storage needed to provide that security function.

The browser receives a short-lived token. Aetheris's server sends that token and, where available, the connecting IP address to Cloudflare for verification. If verification fails, the server does not write the request to Attio. Turnstile is configured as a security control, not as an advertising tool.

Cloudflare's own terms and privacy materials provide more detail about its processing. Provider roles, transfer safeguards and the final legal basis must be confirmed in the required pre-production legal review.

Measurement

What is—and is not—active in this preview

The qualification interface creates a small allowlisted set of lifecycle events in the page's in-memory dataLayer, such as form view, step completion, submission and result. The event payload deliberately excludes names, email addresses, companies, store URLs, free text, campaign values and CRM identifiers.

In this preview build, those event hooks do not by themselves install an analytics or advertising cookie. If Aetheris later connects them to Google Analytics, an advertising platform, a session-replay product or another measurement service, the production inventory, consent mechanism and this notice must be updated before activation.

No statement in this preview draft authorises the installation of non-essential analytics or advertising technologies without the consent assessment required by applicable law.

Controls

How to manage browser data

You can use your browser to inspect, block or remove cookies and site storage. The exact controls depend on the browser and device. Common options include:

  • clearing cookies and site data for aetherisstudio.com;
  • blocking third-party cookies or limiting cross-site tracking;
  • using private browsing for a session that should not retain local storage after it closes; and
  • using browser developer or privacy tools to inspect values stored for the site.

Removing the hero session value may cause the opening sequence to play again. Removing attribution storage clears the remembered first-touch context. Blocking Turnstile or its required scripts can prevent the protected qualification form from being sent; you can still contact Aetheris at info@aetherisstudio.com.
